Investigation loop

How an alert becomes a report

Every alert follows the same path: one planner, several specialists working in parallel, one synthesized report. Nothing waits in a queue longer than it has to.

The roster

Seven agents, one dispatch board

Each agent has a narrow job and a fixed set of actions. Dispatch never does the work itself, it decides who does.

  • Built-in 16

Safety & resilience

  • Disk-backed SQLite event queue (WAL mode)
  • Event dedup with cross-restart persistence
  • Circuit breaker · exponential backoff
  • mTLS + HMAC-signed transport

On-agent detection

  • YARA matcher — 17 rules, pure Go, no C deps
  • PE parser — import table, sections, packer ID
  • XOR cipher detector — single & multi-byte
  • Process ancestry tree — 100k LRU, WAL persistence
  • FP learning — auto-throttles after 10 dismissals

Multi-node

Edge nodes report to one server

Run TRACE at the edge on every host, sync to a central server, and let correlation confidence grow with the number of nodes that agree.

  • Edge nodes sync every 30 seconds
  • 1 node reporting an IOC: 0.5 confidence
  • 2 nodes agree: 0.75 confidence
  • 3+ nodes agree: 0.9 confidence
