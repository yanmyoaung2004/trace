Overview

TRACE is an open-source cybersecurity operations platform designed for security analysts, SOC teams, and DevSecOps engineers. It replaces 4+ traditional security tools — SIEM, SOAR, compliance scanner, EDR console — with a single Go binary that runs on any OS without Docker, Elasticsearch, or any external service.

Key Design Principles

  • Single binary, zero dependencies. No Docker, no Python, no Elasticsearch, no external databases. The binary embeds SQLite, YARA rules, MITRE data, and Wazuh decoders.
  • Offline-first. Local analysis, cache, and playbooks work without internet. External enrichment is asynchronous and non-blocking.
  • Playbook-driven orchestration. Investigations follow YAML playbooks. LLM planning is a fallback for unrecognized intents.
  • SQLite as everything. Task queue, result store, cache, configuration, threat intel — all in one WAL-mode SQLite database.
  • Edge-to-server sync. Deploy agents at network edges; results sync to a central dashboard with cross-node IOC correlation.

Quick Start

git clone https://github.com/yanmyoaung2004/trace.git
cd trace/dev
go build -o trace ./cmd/trace

# Run your first investigation (no setup needed)
./trace investigate "check hash 275a021bbfb6489e54d471899f7db9d1663fc695ec2fe2a2c4538aabf651fd0f"

# Start the SIEM engine
./trace serve --siem --log-dir /var/log

System Architecture

TRACE operates as a multi-agent system where specialized agents collaborate to process security investigations through a SQLite-backed task queue.

Input (CLI / API / SIEM alert)
  └─ Dispatch Agent (orchestrator)
       ├─ Sift Agent (malware analysis, YARA, PE, VT)
       ├─ Archive Agent (MITRE, CVE, intel enrichment)
       ├─ Response Agent (block, quarantine, kill)
       ├─ SCA Agent (compliance scanning)
       ├─ EDR Agent (endpoint monitoring & response)
       ├─ Notifier (Slack, Discord, Telegram)
       └─ Exporter (report serving)
  └─ Investigation Report + Case

Deployment Models

Standalone: Run trace or trace serve on a single machine. All agents, database, and SIEM engine run in one process.

Central Server + Edge: Deploy trace-agent on endpoints. Each edge runs the full monitoring stack and syncs investigations to a central server every 30s.

# Central server
trace server --http-addr :8080

# Edge node
trace-agent --server http://central:8080 --api-key xxx

# On the server
trace edr list

EDR Agent

TRACE ships its own lightweight endpoint agent (trace-agent) — no third-party EDR required. The agent runs on Windows, Linux, and macOS.

CLI

trace-agent [flags]

Flags:
  --config string      Config path (default ~/.trace-agent/config.json)
  --server string      Trace server URL (env: TRACE_AGENT_SERVER)
  --api-key string     API key (env: TRACE_AGENT_API_KEY)
  --install            Install as system service
  --uninstall          Remove system service
  --service            Run as Windows service (SCM)
  --status             Show agent status
  --version            Print version
  --verbose            Enable verbose/debug logging

Resilience

  • Disk-backed SQLite event queue (WAL mode, oldest-eviction)
  • Event dedup with SQLite persistence across restarts
  • Circuit breaker (5 failures → 60s open)
  • mTLS + HMAC-signed transport
  • Auto-update with SHA256 verification + atomic swap
  • Process tree: 100k LRU cap, WAL incremental log, crash replay
  • Local correlation: process burst, rapid deletion, suspicious children, create-then-delete, kill-chain reconstruction
  • FP learning: auto-throttles after 10 dismissals

Management (from Trace server)

trace edr list                                         # List active agents (--all for revoked)
trace edr view <id>                                    # Details: CPU name, memory in GB, IP
trace edr events <id> [--type TYPE] [--min-severity N] # Events with YARA rule name + event ID
trace edr events <id> --type alert --limit 10          # Filter by type, severity, limit
trace edr dispatch <id> block_ip --ip 203.0.113.42     # Block IP
trace edr dispatch <id> kill_process --pid 4521        # Kill process
trace edr dispatch <id> quarantine_file --path C:\mal.exe # Quarantine file
trace edr dispatch <id> isolate_host                    # Isolate from network
trace edr dispatch <id> system_snapshot                 # System snapshot
trace edr dismiss <alert-id> [--reason "note"]         # Mark false positive
trace edr revoke <id>                                  # Remove agent

Threat Intelligence

VirusTotal (hash/URL/IP), AbuseIPDB (IP reputation), AlienVault OTX (indicator pulses), Firecrawl (web search OSINT). Built-in MITRE ATT&CK (81 techniques) + 23 IOC entries.

Notifications

Slack (webhook), Discord (webhook), Telegram (bot API). Configured via trace init or config file.

Terminal UI

5 screens: Dashboard, Investigations, Cases, SIEM, Config. Keyboard navigation (Tab, arrows, Enter, Esc, 1-5). Built with Bubbletea.

Server & API

REST API with Bearer token auth. Endpoints: register, heartbeat, push, nodes, investigations, correlations, timeline, health. Default user auto-seeded with random API key.

Plugin System

Built-in: SCA Agent, Exporter. External: Go shared libraries (.so) loaded from ~/.trace/plugins/. Must implement AgentPlugin interface.

Internationalization

English + Myanmar (Burmese). Auto-detected via LANG/LC_ALL env vars.
