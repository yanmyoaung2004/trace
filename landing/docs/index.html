1. Overview

Trace is an open-source cybersecurity operations platform designed for security analysts, SOC teams, and DevSecOps engineers. It replaces 4+ traditional security tools — SIEM, SOAR, compliance scanner, and EDR console — with a single Go binary that runs on any OS without Docker, Elasticsearch, or any external service.

Core Capabilities

CapabilityDescriptionMetrics
SIEM EngineReal-time log monitoring with 462 detection rules, 1,567 decoders, windowed correlation, and MITRE ATT&CK mapping462 rules, 1567 decoders
SOAR PipelineAlert-to-investigation-to-response automation with playbook-driven workflows and HITL approvals27 playbooks
ComplianceAutomated security configuration assessment against 8 major frameworks with evidence collection and PDF reports8 frameworks
EDRRemote endpoint isolation, scan, kill, and script execution across CrowdStrike, SentinelOne, and MDE3 providers
Threat IntelligenceMulti-source enrichment via VirusTotal, AbuseIPDB, AlienVault OTX, and web search4 providers
Case ManagementFull case lifecycle with timeline, IOCs, evidence attachments, investigation linking, and PDF exportAuto-creation from alerts
Threat HuntingScheduled automated hunts with configurable playbooks and severity thresholds4 default hunts
Agent Architecture7 specialized agents (Dispatch, Sift, Archive, Response, SCA, Notifier, EDR) collaborating via SQLite-backed task queue14 agent capabilities

Key Design Principles

  • Single binary, zero dependencies. Download and run. No Docker, no Python, no Elasticsearch, no external databases. The binary embeds SQLite, YARA rules, MITRE data, and Wazuh decoders.
  • Offline-first. Local analysis, cache, and playbooks work without internet. External enrichment (VT, intel) is asynchronous and non-blocking.
  • Playbook-driven orchestration. Investigations follow YAML playbooks. LLM planning is a fallback for unrecognized intents.
  • SQLite as everything. Task queue, result store, cache, configuration, threat intel — all in one WAL-mode SQLite database.
  • Edge-to-server sync. Deploy agents at network edges; results sync to a central dashboard with cross-node IOC correlation.

4. System Architecture

Trace operates as a multi-agent system where specialized agents collaborate to process security investigations. The platform follows a pipeline architecture: intent in, report out.

High-Level Data Flow

Log Sources ──> SIEM Engine ──> Dispatch Agent ──┬──> Archive Agent
                                                 ├──> Sift Agent
                                                 ├──> Response Agent
                                                 └──> Notifier

The Dispatch Agent is the central orchestrator. It receives intents (from CLI, SIEM alerts, or scheduled hunts), classifies them, selects or plans a playbook, dispatches tasks to specialist agents, collects results, and synthesizes a final report.

Agent Communication

Agents communicate through a SQLite-backed task queue. The Dispatch Agent writes tasks to a tasks table. Specialist agents poll for pending tasks, execute, and write results. All agent outputs are persisted and available for audit, replay, and report generation.

File-Based Artifact Exchange

For large artifacts (malware samples, PCAPs, disk images), agents exchange references to shared files rather than embedding payloads in JSON:

/data/shared/<investigation_id>/
  sift/
    hash_lookup_result.json
    yara_result.json
    vt_result.json
  archive/
    mitre_mapping.json
    cve_details.json
  response/
    block_result.json

Investigation Lifecycle

  1. Intent Received — via CLI, SIEM alert, scheduled hunt, or API
  2. Intent Classification — keyword/regex matching or LLM fallback
  3. Playbook Selection — best-match playbook loaded from built-in or external library
  4. Task Dispatching — steps dispatched to specialist agents asynchronously
  5. Result Collection — agents write results back; pipeline merges partial results
  6. Report Synthesis — Dispatch Agent generates structured report with confidence score
  7. Case Linking — high-severity results auto-create or link to existing cases
  8. Notification — configured channels receive summary

5. Agent System

Trace includes 7 built-in agents, each implementing a standard interface with Name(), Capabilities(), and Execute() methods.

Dispatch Agent

Orchestrator The central planner and orchestrator.

  • Intent classification — keyword/regex matching against structured data; LLM fallback for natural language queries
  • Playbook selection — matches intent to best-fit playbook from built-in or external library
  • LLM planner — supports OpenAI, Anthropic, and Ollama for dynamic plan generation
  • Report synthesis — aggregates all agent results into a structured report with confidence scoring

Sift Agent

Malware Analysis File and binary analysis engine.

  • YARA scanning — 15 built-in rules covering EICAR, PowerShell abuse, packed binaries, process injection, keyloggers, etc.
  • PE analysis — extracts hashes (MD5/SHA1/SHA256), compile timestamp, entry point, section entropy, imports, and suspicious indicators
  • Hash lookup — built-in cache of 9 known malware families (Mimikatz, CobaltStrike, Emotet, Ryuk, PlugX, AgentTesla, WannaCry, etc.)
  • Rootkit detection — 200+ file pattern signatures across 60+ known rootkit families, 45+ trojan binary signatures, 10+ OS behavior checks
  • VirusTotal lookup — hash, URL, and IP lookups with rate limiting

Archive Agent

Threat Intel Threat intelligence and knowledge base.

  • MITRE ATT&CK — 81 techniques with descriptions, tactics, platforms, mitigations, and detection guidance
  • CVE lookup — NVD API v2.0 integration with 24-hour SQLite cache; returns CVSS score, severity, vector, affected CPEs
  • Malware lookup — 23 built-in IOC entries across hashes, IPs, and domains
  • Web search — Firecrawl API for open-source intelligence gathering

Response Agent

Response Host-level response actions.

ActionWindowsLinuxmacOS
block_ipnetsh advfirewalliptablespfctl
quarantine_filemovemvmv
kill_processtaskkillkillkill
restart_servicescsystemctllaunchctl

Every response action generates an audit record with rollback command and status.

SCA Agent

Compliance Security configuration assessment. Runs CIS benchmark policies across Windows, Linux, and macOS. Supports force-scan mode that bypasses requirements for direct policy execution.

Notifier Agent

Notifications Sends alerts and investigation summaries to Slack, Discord, and Telegram via webhooks.

EDR Agent

Endpoint Remote endpoint management via CrowdStrike, SentinelOne, and Microsoft Defender for Endpoint. See EDR Connectors section.

6. Deployment Models

Standalone (Single Node)

Default mode. Run trace or trace serve on a single machine. All agents, database, and SIEM engine run in one process. Ideal for individual analysts, small SOCs, or air-gapped environments.

Central Server + Edge Nodes

Deploy edge nodes at remote sites (branch offices, OT networks, cloud VPCs). Each edge runs the full binary with local SQLite. Edges sync investigations to a central server every 30 seconds.

# Central server (on-prem or cloud)
trace server --http-addr :8080

# Edge node (remote site)
trace serve --server-addr http://central:8080 --siem --log-dir /var/log

The central dashboard provides cross-node IOC correlation with confidence scoring:

Nodes with Same IOCConfidence
1 node0.5 (possible)
2 nodes0.75 (likely)
3+ nodes0.9 (confirmed campaign)

10. SOAR Pipeline

The SOAR (Security Orchestration, Automation, and Response) pipeline connects SIEM alerts to automated investigations, playbook execution, response actions, case creation, and notifications — all within the same binary.

Pipeline Flow

SIEM Alert ──> Auto-Investigation ──> Playbook ──> Report ──> Auto-Case ──> Notification
  1. Alert triggers investigation. When a SIEM rule fires with severity >= 4, Trace automatically creates an investigation
  2. Playbook dispatches. The Dispatch Agent selects the appropriate playbook based on rule mapping
  3. Agents execute. Tasks are dispatched to Sift, Archive, Response, or EDR agents asynchronously
  4. Report generated. Results are synthesized into a structured report with confidence score
  5. Case auto-created. High-severity investigations create or link to security cases
  6. Notification sent. Configured channels receive the investigation summary

Human-in-the-Loop Approval

Playbooks can require analyst approval before executing response actions:

trace approval pending     # List requests waiting for review
trace approval approve <id>  # Approve and execute
trace approval deny <id>     # Deny and roll back

Playbook Format

name: hash-lookup
description: Check a file hash against local cache, YARA, and VirusTotal
steps:
  - agent: sift
    action: hash_lookup
    params:
      hash: ${input.hash}
  - agent: sift
    action: yara_scan
    params:
      path: ${input.path}
  - agent: sift
    action: vt_lookup
    params:
      hash: ${input.hash}
    timeout: 30s
    optional: true
  - agent: dispatch
    action: synthesize_report

Step Features

  • Conditional execution — if field to gate steps on prior results
  • HITL approval — wait: analyst_approval pauses for human review
  • Parameter interpolation — ${input.field} and ${results.step_id.field}
  • Timeouts — per-step timeout with graceful failure
  • Optional steps — optional: true continues pipeline on failure

13. Case Management

Full case lifecycle management with timeline tracking, IOC management, evidence attachments, and PDF export.

Case States

open investigating resolved closed

Severity Levels

low medium high critical

Automation

Cases are automatically created from SIEM alerts with severity >= 4. The case inherits the alert title, severity, and links to the investigation. Manual creation is also supported:

trace case create --title "Phishing campaign Q3" --severity high --description "Targeting finance department"

Evidence Attachments

trace case evidence <case-id> --file malware_sample.exe --name "Suspicious binary" --mime application/octet-stream

Evidence files are stored in the case directory and included in PDF exports.

PDF Export

trace case export-pdf <case-id> -o case_report.pdf

The generated PDF includes: case metadata, timeline, IOCs, linked investigations, and evidence file list. A4 format, professional layout.

20. Plugin System

Trace supports two plugin mechanisms: compiled-in agents (core) and external shared libraries (third-party).

Built-in Plugins

PluginPurpose
SCA AgentSecurity configuration assessment with CIS policy engine
ExporterHTML report server for compliance and investigation reports

External Plugins

  • Go shared library (.so) files loaded from ~/.trace/plugins/
  • Must implement the AgentPlugin interface
  • Plugin registry: https://github.com/yanmyoaung2004/trace/releases/latest/download/plugins/index.json
trace plugin list           # List all registered agents
trace plugin search <query>  # Search plugin registry
trace plugin install <name>  # Download and install
trace plugin remove <name>   # Remove plugin
